# Regression Checklist

## Planning create/edit/view
- Create new planning with valid process/shift/room assignments.
- Edit existing planning and verify cards, stages, and summaries.
- Open planning view and validate stage/device summaries.

## Downtime/overtime hold-resume
- Add downtime and verify hold overlay/state update.
- Resume process and confirm hold UI/state clears.
- Add/remove overtime windows and verify estimation updates.

## Operator task testing lifecycle
- Search device, execute pass/NG flow, and verify status updates.
- Validate sticker/packaging flow and expected stage transitions.
- Confirm retry path does not duplicate incorrect counters.

## Stage-wise attempt history
- Verify attempts count reflects stage-specific history.
- Verify history modal/logs align with current stage and date filters.

## Carton movement transitions
- Add device to carton and validate carton status updates.
- Shift carton to next stage/store and verify inventory/task reflections.
- Confirm error messages are user-friendly when transition fails.
